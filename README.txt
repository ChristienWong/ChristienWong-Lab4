My first feature is a view on web button in the detail view controller. This is a fairly basic feature that takes the movie's ID and sends the user to the official TMDb URL, where they can continue exploring the movie in the app. The feature uses Apple’s SafariServices framework, so the app is able to unwrap the movie object and cast the URL into a View Controller. This creates a fully functional web browser in the app without forcing the user to switch to a standalone Safari Application. 
For my second creative feature, I created a delete all favorites button for the favorites section of the app. I added a trash bar button item to the nav bar so users don’t have to swipe every single row. To prevent accidents, I also used an alert controller with .alert style. IT uses a UI Alert Action and a .destructive delete all button. When the destruction action is confirmed, it empties the local favorites array and uses a JSONEncoder to encode that empty array to overwrite the existing UserDefaults data. Finally, it calls a tableView.reloadData() to clear the UI.The last feature I made was a share button allowing users to send movie recommendations to friends via URL. IT works by featuring UIActivityViewController to use iOS-level sharing capabilities. The app creates an array of Any items with a custom string and url. The array then passes through the Activity Controller and prevents popover crashes using the popoverPresentationController.sourceView. The OS creates a share sheet with actions like messages, mail, and Airdrop based on the user’s apps.
